# Первый запрос Runner'а

При запуске метода `Runner.listen()` выполняется первый "холодный" опрос серверов FunPay для инициализации состояния.

## Алгоритм инициализации

1. **Загрузка списка чатов:** Runner делает запрос к основной странице чатов `https://funpay.com/chat/`.
2. **Парсинг последних сообщений:** Извлекаются последние сообщения для всех доступных чатов (не более 50).
3. **Генерация событий:** Для каждого обнаруженного чата генерируется событие `InitialChatEvent`.
4. **Сохранение состояния:** 
   * Последние сообщения (их текст до 250 символов и время отправки) сохраняются в словарь `Runner.last_messages`.
   * Только тексты сообщений сохраняются в словарь `Runner.init_messages`.

## Структуры данных кэша

### `Runner.last_messages`

Используется для сверки времени и текста последнего сообщения при последующих опросах:

```python
{
    48392847: ["Привет, лот ещё актуален?", "14:32"],
    58392098: ["Спасибо, всё получил!", "Вчера"],
    # ...
}
```

### `Runner.init_messages`

Используется исключительно при генерации первых событий новых сообщений, чтобы отсечь те сообщения, которые были отправлены до запуска скрипта:

```python
{
    48392847: "Привет, лот ещё актуален?",
    58392098: "Спасибо, всё получил!",
    # ...
}
```
