# FunPayAPI — Документация

Неофициальная реализация библиотеки для взаимодействия между API FunPay и вашим Python-клиентом. Вся документация и исходный код базируются на библиотеке от [LIMBODS/FunPayAPI](https://github.com/LIMBODS/FunPayAPI).

Версия библиотеки: 1.1.0

## Установка

Все способы установки и обновления описаны в файле setup.md.

```bash
pip install FunPayAPI
```

## Структура каталога

* docs/Account/ — управление аккаунтом FunPay (сообщения, лоты, заказы).
* docs/Runner/ — бесконечный цикл прослушивания событий.
* docs/Events/ — классы генерируемых событий.
* docs/Types/ — типы данных, используемые библиотекой.
* docs/Enums/ — перечисления для типизации статусов и типов.
* docs/HOW_chats/ — описание алгоритма обнаружения новых сообщений.
* AI_PROMPT.md — подробный системный промпт (правила инициализации Account, задержки Runner, работа с ID заказов) для ИИ-ассистентов.

## Навигация по разделам

### Account
Класс для управления действиями аккаунта — отправка сообщений, картинок, управление лотами и заказами.
* get() — получение информации об аккаунте.
* send_message() / send_image() — отправка контента в чаты.
* get_chat() / get_chat_history() — чтение чатов и истории.
* get_order() / get_orders() — получение информации о заказах.
* raise_lots() — поднятие лотов на торговой площадке.
* refund() — оформление возврата средств покупателю.

### Runner
Класс для прослушивания событий FunPay в реальном времени.
* listen() — блокирующий генератор событий (опрос с настраиваемой задержкой).
* get_updates() / parse_updates() — получение и разбор сырых пакетов обновлений.

### Events
События, возвращаемые генератором Runner.listen():
* BaseEvent — базовый класс события.
* InitialChatEvent / InitialOrderEvent — события первоначального состояния при старте.
* NewMessageEvent — получение нового сообщения от покупателя.
* NewOrderEvent — оплата нового заказа покупателем.
* OrderStatusChangedEvent — изменение статуса заказа (например, подтверждение).
